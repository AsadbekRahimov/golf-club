# Руководство администратора

## 1. Вход в систему

### Доступ к админ-панели
- URL: `https://ваш-домен.com/admin`
- Введите email и пароль администратора

### Первый вход
После установки системы создайте администратора:
```bash
php artisan orchid:admin admin admin@example.com password123
```

---

## 2. Главная панель (Dashboard)

После входа вы увидите панель с ключевыми метриками:

| Метрика | Описание |
|---------|----------|
| Новые заявки | Клиенты, ожидающие подтверждения |
| Клиентов | Общее число подтвержденных клиентов |
| Ожидающих бронирований | Запросы на услуги |
| Чеков на проверку | Платежи с загруженными чеками |
| Активных подписок | Текущие действующие подписки |
| Свободных шкафов | Доступные для аренды шкафы |

---

## 3. Управление клиентами

### 3.1 Новые заявки
**Меню:** Клиенты → Новые заявки

Здесь отображаются клиенты, зарегистрировавшиеся через Telegram-бот и ожидающие подтверждения.

**Действия:**
1. Нажмите на имя клиента для просмотра деталей
2. **Подтвердить** - клиент получит доступ к боту
3. **Отклонить** - клиент получит уведомление об отказе

### 3.2 Все клиенты
**Меню:** Клиенты → Все клиенты

Список всех клиентов с фильтрацией по статусу.

**Статусы:**
- 🟡 **Ожидает подтверждения** - новый клиент
- 🟢 **Подтвержден** - активный клиент
- 🔴 **Заблокирован** - доступ ограничен

### 3.3 Карточка клиента
При открытии клиента доступно:
- Просмотр контактных данных
- Telegram ссылка (если есть username)
- Редактирование ФИО
- Заметки
- Список активных подписок
- Действия: подтвердить/отклонить/заблокировать/разблокировать

---

## 4. Управление бронированиями

### 4.1 Ожидающие бронирования
**Меню:** Бронирования → Ожидающие

Запросы от клиентов, требующие обработки.

**Статусы заявок:**
- 🟡 **Ожидает рассмотрения** - новая заявка
- 🔵 **Требуется оплата** - клиенту отправлены реквизиты
- 🔷 **Чек отправлен** - клиент загрузил чек, ждёт проверки
- 🟢 **Одобрено** - услуги активированы
- 🔴 **Отклонено** - заявка отклонена

### 4.2 Обработка заявки

**Шаг 1:** Откройте заявку, нажав "Обработать"

**Шаг 2:** Выберите действие:

| Действие | Когда использовать |
|----------|-------------------|
| **Подтвердить без оплаты** | VIP-клиент, бесплатная услуга, предоплата наличными |
| **Запросить оплату** | Стандартный процесс - клиент получит реквизиты |
| **Отклонить** | Некорректная заявка, нет свободных шкафов |

**Шаг 3 (если запрошена оплата):**
- Клиент получает сообщение с реквизитами
- Клиент оплачивает и отправляет фото чека в бот
- Заявка переходит в статус "Чек отправлен"
- Вы проверяете чек и нажимаете "Подтвердить оплату"

### 4.3 Что происходит при подтверждении
1. Создаются подписки согласно заявке
2. Если есть шкаф - назначается первый свободный
3. Клиент получает уведомление в Telegram с деталями

---

## 5. Проверка платежей

### 5.1 Проверка чеков
**Меню:** Платежи → Проверка чеков

Список платежей с загруженными чеками, ожидающими проверки.

**Процесс проверки:**
1. Нажмите "Проверить"
2. Откроется карточка заявки с изображением чека
3. Сверьте сумму и реквизиты
4. **Подтвердить оплату** или **Отклонить** (с указанием причины)

### 5.2 История платежей
**Меню:** Платежи → Все платежи

Полная история всех платежей с фильтрацией.

---

## 6. Управление шкафами

**Меню:** Управление → Шкафы

### Просмотр шкафов
Таблица всех шкафов с информацией:
- Номер шкафа
- Статус (свободен/занят)
- Клиент (если занят)
- Дата окончания аренды

### Добавление шкафов
1. Введите количество в поле "Количество новых шкафов"
2. Нажмите "Добавить шкафы"
3. Шкафы создадутся с автоматической нумерацией

---

## 7. Управление подписками

**Меню:** Управление → Подписки

### Просмотр подписок
Список всех подписок с информацией:
- Клиент
- Тип подписки
- Номер шкафа (если есть)
- Даты начала и окончания
- Статус

### Отмена подписки
1. Найдите подписку в списке
2. Нажмите "Отменить"
3. Подписка станет неактивной
4. Шкаф (если был) освободится автоматически

### Метки
- **Истекает** - подписка заканчивается в ближайшие 3 дня

## 9. Автоматические процессы

Система автоматически выполняет:

| Время | Действие |
|-------|----------|
| 09:00 | Уведомление клиентов об истекающих подписках |
| 00:05 | Обработка истекших подписок, освобождение шкафов |

## 11. Типичные сценарии

### Сценарий: Новый клиент
```
1. Клиент пишет /start в боте
2. Клиент отправляет номер телефона
3. В админке появляется в "Новые заявки"
4. Админ подтверждает клиента
5. Клиент получает доступ к боту
```

### Сценарий: Бронирование с оплатой
```
1. Клиент выбирает услугу в боте
2. В админке появляется в "Ожидающие бронирования"
3. Админ нажимает "Запросить оплату"
4. Клиент получает реквизиты и отправляет чек
5. В админке появляется в "Проверка чеков"
6. Админ проверяет и подтверждает оплату
7. Подписки активируются, клиент уведомлён
```

### Сценарий: Истечение подписки
```
1. За 3 дня до истечения - клиент получает напоминание
2. В день истечения - подписка становится неактивной
3. Шкаф (если был) освобождается автоматически
```
